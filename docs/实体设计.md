# Process Management Tool - Entity Design (Phase 1)

## Architecture Philosophy
- Treat tasks as "Processes" (running, blocked, suspended, terminated).
- Keep entities minimalist to ensure a high ceiling for future features (Heatmaps, Timelines, PKM exports).
- Logs are append-only time streams.

## 1. Entity: `Process`
Represents a working thread or a fuzzy goal. Supports infinite hierarchy via `parent_id`.

| Column        | Type         | GORM Tags / Constraints                   | Description                                      |
| ------------- | ------------ | ----------------------------------------- | ------------------------------------------------ |
| `id`          | INTEGER      | `primaryKey;autoIncrement`                | Process ID (PID)                                 |
| `parent_id`   | INTEGER      | `index;default:null`                      | Parent PID (for sub-processes)                   |
| `title`       | VARCHAR(255) | `not null`                                | Name of the process                              |
| `description` | TEXT         | `default:null`                            | Markdown description of the overall goal         |
| `status`      | VARCHAR(20)  | `index;default:'running'`                 | `running`, `blocked`, `suspended`, `terminated`  |
| `priority`    | VARCHAR(20)  | `default:'medium'`                        | `low`, `medium`, `high`                          |
| `ranking`     | REAL         | `index;default:0.0`                       | Custom sorting weight (like UNIX nice value)     |
| `created_at`  | DATETIME     | `autoCreateTime`                          | When the process was created                     |
| `updated_at`  | DATETIME     | `autoUpdateTime`                          | Automatically updated on any process change      |

## 2. Entity: `Log`
A universal append-only time stream attached to a process. Can be user notes, plans, or system-generated state changes.

| Column        | Type         | GORM Tags / Constraints                   | Description                                      |
| ------------- | ------------ | ----------------------------------------- | ------------------------------------------------ |
| `id`          | INTEGER      | `primaryKey;autoIncrement`                | Log ID                                           |
| `process_id`  | INTEGER      | `index;not null`                          | The PID this log belongs to                      |
| `content`     | TEXT         | `not null`                                | Markdown content (notes, plans, state changes)   |
| `created_at`  | DATETIME     | `autoCreateTime;index`                    | Crucial for Timelines and Heatmaps               |

## Database Requirements for Claude Code
1. Use `gorm` with pure-Go `glebarez/sqlite`.
2. Ensure `parent_id` in `Process` correctly sets up a self-referencing relation if needed, though simple integer querying is fine.
3. Enable SQLite **FTS5 (Full-Text Search)** extension. Create a virtual table to index `Process.title`, `Process.description`, and `Log.content` for instant global search.
